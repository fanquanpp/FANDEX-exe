# 贡献指南

感谢你对 FANDEX 项目的关注。本指南旨在帮助你了解如何为本项目贡献内容。

## 内容来源说明

本仓库所有内容均由人工与人工智能（AI）共同编写、搜集、整理与编排。我们欢迎任何形式的贡献，包括但不限于：

- 修正文档中的错误或不准确之处
- 补充遗漏的知识点或示例
- 更新过时的技术信息
- 改进文档结构与表述
- 新增模块或文档

## 内容质量要求

### 准确性

- 所有技术内容应基于官方文档或权威资料
- 代码示例应经过验证，确保可运行
- 引用第三方信息时，应注明来源

### 零基础友好

- 文档应面向零基础学习者，避免假设读者已有先修知识
- 首次出现的专业术语应给出简要解释
- 代码示例应包含中文注释，并附带预期输出
- 每篇文档应包含"常见错误与排查"章节

### 排版规范

- 中英文之间必须加半角空格
- 中文与数字之间必须加半角空格
- 统一使用中文标点（代码和链接除外）
- 不使用 emoji
- 标题层级不超过三级
- 代码块必须标注语言类型

### 术语统一

- Agent：首次出现写"智能体（Agent）"，后续统一用"智能体"
- API：统一保留英文缩写
- Gateway：统一使用"网关"

## AI 协作说明

本仓库内容由人工与 AI 共同编撰，贡献者在提交内容时应注意：

- AI 生成的内容应经过人工审核与验证
- 不应直接提交未经核实的 AI 输出
- AI 辅助编写的内容应确保技术准确性
- 鼓励使用 AI 工具提升效率，但最终质量由提交者负责

## 提交规范

- 提交信息使用简体中文
- 每次提交应聚焦于单一变更
- 提交前运行 `npm run build` 确保构建通过
- 提交前运行 `npm run lint:docs` 检查文档格式

## 自学能力倡导

本仓库鼓励学习者在学习过程中培养以下能力：

- **自主检索**：学会使用搜索引擎、官方文档、AI 工具高效获取信息
- **交叉验证**：对获取的信息进行多来源比对，确认准确性
- **批判性思考**：不盲信任何单一信息来源，包括本仓库内容
- **持续学习**：技术发展迅速，保持学习与更新的习惯

## 问题反馈

如发现内容错误或有改进建议，欢迎通过以下方式反馈：

- 提交 GitHub Issue
- 提交 Pull Request
- 在相关文档页面留言

感谢你的贡献。
