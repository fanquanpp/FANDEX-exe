## A

| 术语       | 英文                         | 释义                                                                     |
| ---------- | ---------------------------- | ------------------------------------------------------------------------ |
| ABI        | Application Binary Interface | 应用二进制接口，定义编译后的程序在二进制层面的调用约定和数据布局         |
| AOV网      | Activity On Vertex           | 顶点表示活动的有向图，拓扑排序确定活动执行顺序                           |
| AOE网      | Activity On Edge             | 边表示活动的有向图，用于关键路径分析                                     |
| 抽象层级   | Abstraction Layer            | 从晶体管到应用的分层模型，每层定义接口契约并隐藏实现细节                 |
| 自动机理论 | Automata Theory              | 研究抽象计算模型（DFA/NFA/图灵机）的理论，是编译原理和计算理论的数学基础 |
| 自动机     | Automaton                    | 根据输入在状态间转移的抽象计算模型，包括有限自动机、下推自动机、图灵机   |

## B

| 术语     | 英文              | 释义                                                                   |
| -------- | ----------------- | ---------------------------------------------------------------------- |
| 布尔代数 | Boolean Algebra   | 基于真值0和1的代数系统，是数字逻辑设计的数学基础                       |
| 总线     | Bus               | 连接CPU、内存和I/O设备的共享通信通道，分为地址总线、数据总线、控制总线 |
| 分支预测 | Branch Prediction | CPU预测条件分支走向以减少流水线气泡的技术，预测失败导致流水线冲刷      |

## C

| 术语           | 英文                 | 释义                                                                                 |
| -------------- | -------------------- | ------------------------------------------------------------------------------------ |
| 缓存一致性     | Cache Coherence      | 多核处理器中保证各核缓存数据一致的协议，如MESI协议                                   |
| Chomsky层次    | Chomsky Hierarchy    | 将形式语言分为0型(递归可枚举)、1型(上下文有关)、2型(上下文无关)、3型(正则)的层次体系 |
| 丘奇-图灵论题  | Church-Turing Thesis | 任何"可有效计算"的函数都可以被图灵机计算的论题，是计算理论的基础假设                 |
| 编译器         | Compiler             | 将源代码翻译为目标代码的程序，分为前端(词法/语法/语义分析)和后端(优化/代码生成)      |
| 复杂性类       | Complexity Class     | 按计算资源需求对问题分类，如P、NP、NP-Complete、NP-Hard、PSPACE                      |
| 上下文无关文法 | Context-Free Grammar | 产生式左部为单个非终结符的文法，2型文法，是大多数编程语言语法分析的基础              |
| CPU调度        | CPU Scheduling       | 操作系统决定哪个就绪进程获得CPU使用权的机制，常见算法有FCFS、SJF、RR                 |
| 临界区         | Critical Section     | 访问共享资源的代码段，同一时刻只允许一个进程/线程进入                                |

## D

| 术语     | 英文                           | 释义                                                                   |
| -------- | ------------------------------ | ---------------------------------------------------------------------- |
| 死锁     | Deadlock                       | 两个或多个进程互相等待对方释放资源而永久阻塞的状态                     |
| 设备驱动 | Device Driver                  | 操作系统中管理特定硬件设备的内核模块，提供统一的设备访问接口           |
| DFA      | Deterministic Finite Automaton | 确定性有限自动机，每个状态对每个输入有唯一转移，与NFA等价              |
| 离散数学 | Discrete Mathematics           | 研究离散结构的数学分支，包括逻辑、集合、关系、图论、组合计数、代数结构 |
| DNS      | Domain Name System             | 将域名映射为IP地址的分布式命名系统，采用层次化递归/迭代查询            |
| 动态链接 | Dynamic Linking                | 程序运行时才将库代码与程序链接的机制，共享库(.so/.dll)节省内存         |

## E

| 术语     | 英文                  | 释义                                                       |
| -------- | --------------------- | ---------------------------------------------------------- |
| 早期发射 | Early Emit / Pipeline | 流水线中指令在完成前就开始下一条指令执行的技术，提高吞吐率 |

## F

| 术语       | 英文                 | 释义                                                                     |
| ---------- | -------------------- | ------------------------------------------------------------------------ |
| 文件系统   | File System          | 操作系统中管理持久数据的子系统，提供文件的创建、读写、删除等操作         |
| 文件控制块 | FCB / Inode          | 文件系统中存储文件元数据(权限、大小、位置)的数据结构                     |
| 有限状态机 | Finite State Machine | 有限个状态之间根据输入进行转移的数学模型，FSM=(Q, Sigma, delta, q0, F)   |
| Fork       | Fork                 | Unix系统调用，创建当前进程的副本(子进程)，子进程获得父进程地址空间的副本 |
| 形式语言   | Formal Language      | 由字母表上满足特定规则的字符串组成的集合，由文法生成                     |

## G

| 术语     | 英文                      | 释义                                                   |
| -------- | ------------------------- | ------------------------------------------------------ |
| 语法分析 | Syntax Analysis / Parsing | 编译器前端阶段，根据文法规则将Token流组织为语法树(AST) |
| 门       | Logic Gate                | 实现布尔运算的基本数字电路元件，包括与门、或门、非门等 |

## H

| 术语     | 英文                        | 释义                                                         |
| -------- | --------------------------- | ------------------------------------------------------------ |
| 哈佛体系 | Harvard Architecture        | 指令和数据存储在不同存储器中的体系结构，可同时取指和取数     |
| 停机问题 | Halting Problem             | 判断任意程序在给定输入上是否停机的问题，已被证明不可判定     |
| HTTP     | HyperText Transfer Protocol | Web应用层协议，基于请求-响应模型，定义了资源获取和提交的方式 |
| 超线程   | Hyper-Threading             | 在一个物理核上模拟两个逻辑核的技术，提高资源利用率           |

## I

| 术语     | 英文                         | 释义                                                                |
| -------- | ---------------------------- | ------------------------------------------------------------------- |
| ISA      | Instruction Set Architecture | 指令集体系结构，定义了软件和硬件之间的接口，如x86-64、ARMv8、RISC-V |
| 中断     | Interrupt                    | 硬件或软件发出的信号，使CPU暂停当前执行转去处理紧急事件             |
| 接口契约 | Interface Contract           | 每个抽象层向上提供的功能承诺，只要接口不变下层实现可任意替换        |

## J

| 术语       | 英文               | 释义                                                         |
| ---------- | ------------------ | ------------------------------------------------------------ |
| 进程       | Process            | 操作系统资源分配的基本单位，拥有独立的地址空间和资源         |
| 进程控制块 | PCB                | 操作系统中存储进程元数据(状态、寄存器、内存信息)的数据结构   |
| 进程调度   | Process Scheduling | 操作系统从就绪队列中选择进程分配CPU的机制                    |
| 进程间通信 | IPC                | 进程之间交换数据的机制，包括管道、消息队列、共享内存、信号量 |

## K

| 术语     | 英文          | 释义                                                         |
| -------- | ------------- | ------------------------------------------------------------ |
| 内核     | Kernel        | 操作系统的核心部分，运行在内核态，管理硬件资源和提供系统服务 |
| 关键路径 | Critical Path | AOE网中从源点到汇点的最长路径，决定工程最短完成时间          |
| 关键字   | Keyword       | 编程语言预定义的保留字，不可用作标识符                       |

## L

| 术语       | 英文             | 释义                                                                    |
| ---------- | ---------------- | ----------------------------------------------------------------------- |
| Lambda演算 | Lambda Calculus  | Alonzo Church提出的函数式计算模型，与图灵机等价，是函数式编程的理论基础 |
| 词法分析   | Lexical Analysis | 编译器前端第一阶段，将字符流分解为Token流，通常用DFA实现                |
| 链接器     | Linker           | 将多个目标文件和库合并为可执行文件的程序，解析符号引用和地址重定位      |

## M

| 术语     | 英文              | 释义                                                                    |
| -------- | ----------------- | ----------------------------------------------------------------------- |
| 宏内核   | Monolithic Kernel | 所有操作系统服务运行在内核态的架构，如Linux，性能高但耦合度高           |
| 微内核   | Microkernel       | 仅最基本服务在内核态，其他服务在用户态的架构，如seL4，安全性高          |
| 内存管理 | Memory Management | 操作系统管理物理和虚拟内存的子系统，包括分页、分段、虚拟内存            |
| 内存映射 | Memory Map / mmap | 将文件或设备映射到进程地址空间的机制，支持高效文件访问                  |
| MESI协议 | MESI Protocol     | 多核缓存一致性协议，定义Modified/Exclusive/Shared/Invalid四种缓存行状态 |
| 互斥     | Mutual Exclusion  | 保证同一时刻只有一个进程/线程进入临界区的同步机制                       |

## N

| 术语   | 英文                              | 释义                                                           |
| ------ | --------------------------------- | -------------------------------------------------------------- |
| NFA    | Nondeterministic Finite Automaton | 非确定性有限自动机，一个状态对同一输入可有多个转移，与DFA等价  |
| 网络层 | Network Layer                     | OSI模型第3层，负责数据包从源到目的端的转发和路由，核心协议为IP |

## O

| 术语     | 英文             | 释义                                                     |
| -------- | ---------------- | -------------------------------------------------------- |
| 操作系统 | Operating System | 硬件与应用之间的中间层，提供进程、虚拟内存、文件三大抽象 |
| OSI模型  | OSI Model        | 开放系统互连七层参考模型，从物理层到应用层的分层网络架构 |
| 优化     | Optimization     | 编译器后端对中间代码进行等价变换以提高执行效率的过程     |

## P

| 术语       | 英文                  | 释义                                                               |
| ---------- | --------------------- | ------------------------------------------------------------------ |
| 流水线     | Pipeline              | 将指令执行分为取指、译码、执行、访存、写回等阶段重叠执行的技术     |
| 分页       | Paging                | 将虚拟地址空间划分为固定大小的页，按需映射到物理页框的内存管理方案 |
| 物理层     | Physical Layer        | OSI模型第1层，定义电气信号、光纤等物理传输介质的规范               |
| 多态       | Polymorphism          | 同一操作作用于不同对象产生不同行为的特性，面向对象核心概念之一     |
| 进程状态机 | Process State Machine | 进程在创建、就绪、运行、阻塞、终止等状态间转移的模型               |
| 信号量     | Semaphore             | Dijkstra提出的同步原语，用整数计数器控制对共享资源的访问           |

## Q

| 术语     | 英文                | 释义                                                                 |
| -------- | ------------------- | -------------------------------------------------------------------- |
| 请求分页 | Demand Paging       | 仅在访问页面时才将其调入内存的虚拟内存管理策略                       |
| 命题逻辑 | Propositional Logic | 以命题为基本单位的逻辑系统，包含否定、合取、析取、蕴含、等价等联结词 |

## R

| 术语       | 英文                             | 释义                                                              |
| ---------- | -------------------------------- | ----------------------------------------------------------------- |
| RISC       | Reduced Instruction Set Computer | 精简指令集计算机，指令简单规整，便于流水线执行，如ARM、RISC-V     |
| 路由       | Routing                          | 网络层功能，选择数据包从源到目的端的传输路径，常见协议有OSPF、BGP |
| 正则表达式 | Regular Expression               | 描述正则语言的表达式，与DFA/NFA等价，广泛用于文本匹配和词法分析   |
| 正则语言   | Regular Language                 | 3型语言，可被DFA/NFA识别，可用正则表达式描述                      |

## S

| 术语         | 英文                  | 释义                                                                     |
| ------------ | --------------------- | ------------------------------------------------------------------------ |
| 系统调用     | System Call           | 用户程序请求操作系统服务的接口，是用户态进入内核态的唯一合法途径         |
| 语义分析     | Semantic Analysis     | 编译器前端阶段，检查类型一致性、作用域规则等语义约束，构建符号表         |
| SOLID原则    | SOLID Principles      | 面向对象设计的五个基本原则：单一职责、开闭、里氏替换、接口隔离、依赖反转 |
| 设计模式     | Design Pattern        | 面向对象软件设计中可复用的解决方案，分为创建型、结构型、行为型           |
| 单一职责原则 | Single Responsibility | 一个类只有一个引起变化的原因，每个类只做一件事                           |
| 开闭原则     | Open/Closed           | 对扩展开放对修改关闭，通过抽象和多态实现                                 |
| 里氏替换原则 | Liskov Substitution   | 子类对象必须能替换父类对象而不破坏正确性                                 |
| 套接字       | Socket                | 网络通信的端点抽象，提供应用层到传输层的接口                             |
| 信号         | Signal                | Unix系统中进程间异步通知机制，如SIGINT、SIGKILL                          |
| 自旋锁       | Spinlock              | 忙等待的锁实现，适用于临界区执行时间极短的场景                           |
| 存储程序     | Stored Program        | 冯诺依曼体系的核心思想，指令和数据存储在同一存储器中                     |

## T

| 术语      | 英文                          | 释义                                                                   |
| --------- | ----------------------------- | ---------------------------------------------------------------------- |
| TCP       | Transmission Control Protocol | 面向连接的可靠传输层协议，通过三次握手建立连接，提供流量控制和拥塞控制 |
| TCP状态机 | TCP State Machine             | TCP连接在各状态(CLOSED/SYN_SENT/ESTABLISHED等)间转移的模型             |
| 线程      | Thread                        | CPU调度的基本单位，同一进程的线程共享地址空间和资源                    |
| 三次握手  | Three-way Handshake           | TCP连接建立过程，客户端发送SYN，服务端回复SYN+ACK，客户端确认ACK       |
| 令牌      | Token                         | 词法分析中将字符流按规则切分后的最小语法单位                           |
| 传递闭包  | Transitive Closure            | 判断图中任意两点是否可达的布尔矩阵，可用Floyd-Warshall求解             |
| 传输层    | Transport Layer               | OSI模型第4层，提供端到端的数据传输服务，核心协议为TCP和UDP             |

## U

| 术语 | 英文                   | 释义                                                   |
| ---- | ---------------------- | ------------------------------------------------------ |
| UDP  | User Datagram Protocol | 无连接的不可靠传输层协议，开销小速度快，适用于实时应用 |

## V

| 术语         | 英文           | 释义                                                               |
| ------------ | -------------- | ------------------------------------------------------------------ |
| 虚拟内存     | Virtual Memory | 操作系统提供的地址空间间接映射机制，每个进程拥有独立的虚拟地址空间 |
| 虚拟文件系统 | VFS            | 操作系统中为不同文件系统提供统一接口的抽象层                       |

## W

| 术语     | 英文          | 释义                                                           |
| -------- | ------------- | -------------------------------------------------------------- |
| 写时复制 | Copy-on-Write | fork时子进程共享父进程页面，仅在实际修改时才复制页面的优化技术 |

## X

| 术语   | 英文               | 释义                                                                 |
| ------ | ------------------ | -------------------------------------------------------------------- |
| 信息论 | Information Theory | Claude Shannon创立的数学理论，量化信息的度量(熵)和传输极限(信道容量) |
| 协议栈 | Protocol Stack     | 网络通信中分层协议的集合，每层添加头部/尾部形成封装-传输-解封流水线  |

## Y

| 术语     | 英文             | 释义                                                             |
| -------- | ---------------- | ---------------------------------------------------------------- |
| 页面替换 | Page Replacement | 物理内存不足时选择淘汰哪个页面的策略，常见算法有LRU、FIFO、Clock |

## Z

| 术语         | 英文                        | 释义                                                             |
| ------------ | --------------------------- | ---------------------------------------------------------------- |
| 中间表示     | Intermediate Representation | 编译器前端与后端之间的中间代码表示，如三地址码、SSA形式          |
| 子网         | Subnet                      | 将IP网络进一步划分的更小网络单元，通过子网掩码确定网络和主机部分 |
| 状态机主线   | State Machine Thread        | 贯穿CS的核心主线之一，关注动态行为如何建模为状态转移图           |
| 体系结构主线 | Architecture Thread         | 贯穿CS的核心主线之一，关注硬件资源如何被组织、寻址、调度         |
