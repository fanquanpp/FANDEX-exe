## React 术语表

## C

| 术语    | 英文      | 释义                                                             |
| ------- | --------- | ---------------------------------------------------------------- |
| Context | Context   | React 提供的跨组件层级传递数据的机制，避免逐层手动传递 props     |
| 组件    | Component | React 应用的基本构建单元，接收 props 并返回描述 UI 的 React 元素 |

## D

| 术语     | 英文           | 释义                                                 |
| -------- | -------------- | ---------------------------------------------------- |
| 代码分割 | Code Splitting | 将应用代码按需拆分为多个包的技术，减少初始加载体积   |
| 调度器   | Scheduler      | React 内部协调更新优先级的模块，决定何时执行渲染工作 |

## E

| 术语   | 英文      | 释义                                                        |
| ------ | --------- | ----------------------------------------------------------- |
| Effect | useEffect | 用于处理副作用的 Hook，在组件渲染后执行订阅、数据获取等操作 |

## F

| 术语  | 英文   | 释义                                                 |
| ----- | ------ | ---------------------------------------------------- |
| Fiber | Fiber  | React 16+ 的协调引擎架构，支持可中断渲染和优先级调度 |
| Ref   | useRef | 用于持有跨渲染周期持久引用的 Hook，不触发重渲染      |

## G

| 术语 | 英文  | 释义                              |
| ---- | ----- | --------------------------------- |
| 挂载 | Mount | 组件实例被创建并插入 DOM 树的过程 |

## H

| 术语     | 英文            | 释义                                                        |
| -------- | --------------- | ----------------------------------------------------------- |
| Hooks    | Hooks           | React 16.8 引入的函数，允许函数组件使用状态和生命周期等特性 |
| 合成事件 | Synthetic Event | React 封装的原生浏览器事件对象，提供跨浏览器一致的事件接口  |

## J

| 术语 | 英文           | 释义                                                              |
| ---- | -------------- | ----------------------------------------------------------------- |
| JSX  | JavaScript XML | JavaScript 的语法扩展，允许在 JS 中编写类似 HTML 的声明式 UI 描述 |

## K

| 术语     | 英文                 | 释义                                                                       |
| -------- | -------------------- | -------------------------------------------------------------------------- |
| 可控组件 | Controlled Component | 表单元素的值由 React state 控制的组件，通过 value 和 onChange 实现双向绑定 |

## L

| 术语   | 英文         | 释义                                                         |
| ------ | ------------ | ------------------------------------------------------------ |
| 懒加载 | Lazy Loading | 通过 React.lazy 和 Suspense 实现的按需加载组件，优化首屏性能 |

## M

| 术语 | 英文    | 释义                                                                 |
| ---- | ------- | -------------------------------------------------------------------- |
| 模式 | Pattern | React 中常用的设计模式，如 Render Props、HOC、Compound Components 等 |

## P

| 术语     | 英文     | 释义                                                                |
| -------- | -------- | ------------------------------------------------------------------- |
| Provider | Provider | Context 的数据提供组件，将值向下传递给所有订阅该 Context 的后代组件 |

## R

| 术语    | 英文           | 释义                                                          |
| ------- | -------------- | ------------------------------------------------------------- |
| Reducer | useReducer     | 管理复杂状态逻辑的 Hook，通过 dispatch 分发 action 来更新状态 |
| 协调    | Reconciliation | React 比较新旧虚拟 DOM 树并计算最小变更的算法过程             |

## S

| 术语              | 英文              | 释义                                                             |
| ----------------- | ----------------- | ---------------------------------------------------------------- |
| Server Components | Server Components | 在服务端渲染且不发送 JavaScript 到客户端的组件，减少客户端包体积 |
| Suspense          | Suspense          | 声明式等待异步数据加载的组件，显示 fallback 直到子组件就绪       |
| 状态提升          | Lifting State Up  | 将共享状态移至最近公共父组件的管理模式，实现子组件间状态同步     |
| 渲染提交          | Commit Phase      | React 将变更实际应用到 DOM 的阶段，发生在协调阶段之后            |

## T

| 术语 | 英文           | 释义                                                                    |
| ---- | -------------- | ----------------------------------------------------------------------- |
| 调和 | Reconciliation | React diff 算法比较前后两棵 Fiber 树差异的过程，确定需要更新的 DOM 节点 |

## V

| 术语     | 英文        | 释义                                                           |
| -------- | ----------- | -------------------------------------------------------------- |
| 虚拟 DOM | Virtual DOM | React 在内存中维护的轻量级 DOM 表示，用于批量计算最小 DOM 操作 |

## W

| 术语 | 英文    | 释义                                                  |
| ---- | ------- | ----------------------------------------------------- |
| 卸载 | Unmount | 组件实例从 DOM 树中移除的过程，执行清理操作如取消订阅 |

## X

| 术语 | 英文   | 释义                                                                 |
| ---- | ------ | -------------------------------------------------------------------- |
| 信号 | Signal | 响应式状态原语，自动追踪依赖并在值变化时触发更新，React 正在探索引入 |

## Y

| 术语     | 英文        | 释义                                                       |
| -------- | ----------- | ---------------------------------------------------------- |
| 严格模式 | Strict Mode | 开发环境下的辅助检查模式，帮助识别不安全的生命周期和副作用 |

## Z

| 术语    | 英文       | 释义                                                                  |
| ------- | ---------- | --------------------------------------------------------------------- |
| 中间件  | Middleware | 在 dispatch 和 reducer 之间插入的处理逻辑，如 redux-thunk、redux-saga |
| 水合    | Hydration  | 服务端渲染的 HTML 在客户端被 React 接管并附加事件处理的过程           |
| Actions | Actions    | React 19 中 Server Functions 的概念，用于处理表单提交和数据变更操作   |
