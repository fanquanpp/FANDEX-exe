1. 自动目录概述

1.1 为什么需要自动目录

自动目录（Table of Contents, TOC）根据文档标题自动生成导航结构，核心价值：

  • 快速导航：点击目录项跳转到对应章节
  • 文档概览：一目了然地了解文档结构
  • 自动更新：标题变更时目录自动同步

1.2 实现方式

方式 语法 适用平台
[TOC] [TOC] 部分编辑器
[[toc]] [[toc]] VuePress 等
{:toc} {:toc} Jekyll/Kramdown
Pandoc --toc 参数 Pandoc
HTML 手动 锚点链接 通用

2.4 Hugo

Hugo 使用模板变量自动生成目录：

2.5 Pandoc

# 生成带目录的文档
pandoc input.md -o output.pdf --toc

# 自定义目录标题
pandoc input.md -o output.pdf --toc -V toc-title="目录"

# 设置目录深度
pandoc input.md -o output.pdf --toc --toc-depth=3

3.2 GitHub 锚点规则

GitHub 自动为标题生成锚点，规则如下：

规则 示例标题 生成的锚点
转小写 Getting Started #getting-started
空格变连字符 Hello World #hello-world
移除特殊字符 What's New? #whats-new
中文保留 安装指南 #安装指南
多个连字符合并 A -- B #a----b

3.3 自动生成工具

# 使用 markdown-toc 自动生成
npx markdown-toc README.md --insert

# 使用 doctoc
npx doctoc README.md

- [概述](#概述)

4.3 折叠目录

使用 HTML <details> 标签创建可折叠目录： 

<details>
<summary> 目录</summary>

- [概述](#概述)
- [安装](#安装)
- [使用](#使用)

</details>

6. 最佳实践

6.1 目录放置位置

  • 文档开头：最常见，方便读者快速导航
  • 固定侧边栏：长文档推荐，始终可见
  • 折叠：中等长度文档，不占用太多空间

6.2 维护建议

  • 使用自动生成工具，避免手动维护
  • 标题命名要清晰，目录才有意义
  • 控制标题层级不超过 4 级
  • 长文档（超过 5 个章节）建议添加目录
