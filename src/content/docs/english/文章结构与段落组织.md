## 1. 文章的基本结构

### 1.1 三段式结构

$$\text{Introduction (开头段)} \rightarrow \text{Body (主体段)} \rightarrow \text{Conclusion (结尾段)}$$

| 部分   | 功能               | 占比   | 要素                       |
| ------ | ------------------ | ------ | -------------------------- |
| 开头段 | 引入主题，提出论点 | 10-15% | 引子 + 背景 + 主题句       |
| 主体段 | 展开论证，提供论据 | 70-80% | 主题句 + 论据 + 分析       |
| 结尾段 | 总结全文，升华主题 | 10-15% | 重述论点 + 总结要点 + 展望 |

### 1.2 段落的基本结构

一个标准段落遵循 **TEEL** 结构：

| 要素 | 英文           | 功能                       |
| ---- | -------------- | -------------------------- |
| T    | Topic Sentence | 主题句：点明段落主旨       |
| E    | Evidence       | 论据：提供事实/数据/引用   |
| E    | Explanation    | 分析：解释论据如何支持主题 |
| L    | Link           | 衔接：连接下一段或回扣论点 |

## 2. 开头段 (Introduction)

### 2.1 开头段的功能

1. **吸引读者注意** (Hook)
2. **提供背景信息** (Background)
3. **提出中心论点** (Thesis Statement)

### 2.2 引子的类型

| 类型     | 方法               | 示例                                                                                         |
| -------- | ------------------ | -------------------------------------------------------------------------------------------- |
| 数据引子 | 引用惊人数据       | Over 4 billion people now use the internet.                                                  |
| 问题引子 | 提出引人思考的问题 | What would happen if AI could think like humans?                                             |
| 引用引子 | 引用名人名言       | As Steve Jobs once said, "Innovation distinguishes between a leader and a follower."         |
| 对比引子 | 呈现对比观点       | While some argue that technology isolates us, others believe it connects us.                 |
| 故事引子 | 讲述简短故事       | In 1969, a team of engineers sent the first message over ARPANET.                            |
| 定义引子 | 定义关键概念       | Machine learning, a subset of artificial intelligence, enables computers to learn from data. |

### 2.3 主题句 (Thesis Statement)

主题句是开头段的核心，明确表达文章的中心论点：

**好的主题句的特征：**

| 特征     | 说明           | 示例                                                                                        |
| -------- | -------------- | ------------------------------------------------------------------------------------------- |
| 明确性   | 清晰表达观点   | Social media has fundamentally changed how people communicate.                              |
| 可论证性 | 可以被论据支持 | Communication is important. (太宽泛)                                                        |
| 具体性   | 包含具体要点   | Social media has changed communication by making it faster, more global, and less personal. |
| 独特性   | 体现独特观点   | Technology is good. (太笼统)                                                                |

**主题句的位置：** 通常在开头段的最后一句。

### 2.4 开头段模板

**议论型：**

> [Hook] In the past decade, artificial intelligence has transformed numerous industries. [Background] From healthcare to finance, AI-powered systems are being deployed at an unprecedented rate. [Thesis] While AI offers significant benefits in efficiency and accuracy, it also raises critical concerns about privacy, employment, and ethical decision-making.

**说明型：**

> [Hook] The internet connects over half of the world's population. [Background] Since its commercialization in the 1990s, the internet has evolved from a simple information-sharing tool to a complex ecosystem of services. [Thesis] This essay examines the evolution of internet technology, its impact on society, and the challenges it presents for the future.

## 3. 主体段 (Body Paragraphs)

### 3.1 主体段的组织方式

| 组织方式   | 说明           | 适用场景     |
| ---------- | -------------- | ------------ |
| 时间顺序   | 按时间先后排列 | 叙述事件发展 |
| 空间顺序   | 按空间位置排列 | 描述场景     |
| 重要性递增 | 从次要到重要   | 论证观点     |
| 重要性递减 | 从重要到次要   | 新闻写作     |
| 对比对照   | 比较异同       | 分析两种方案 |
| 因果关系   | 分析原因和结果 | 解释现象     |

### 3.2 主题句的写法

**主题句 = 段落主旨 + 与论点的联系**

| 类型   | 示例                                                                               |
| ------ | ---------------------------------------------------------------------------------- |
| 论点型 | One significant advantage of cloud computing is its scalability.                   |
| 对比型 | Unlike traditional databases, NoSQL solutions offer flexible schema design.        |
| 因果型 | The rapid growth of e-commerce has led to increased demand for logistics services. |
| 问题型 | One major challenge in deep learning is the need for large labeled datasets.       |

### 3.3 论据的类型

| 论据类型 | 说明           | 示例                              |
| -------- | -------------- | --------------------------------- |
| 事实     | 客观存在的事实 | Python was released in 1991.      |
| 数据     | 统计数字       | The market grew by 25% in 2025.   |
| 引用     | 权威人士的观点 | According to a Nature study, ...  |
| 例子     | 具体案例       | For instance, Google's AlphaGo... |
| 类比     | 类比说明       | Like a pipeline in a factory, ... |
| 逻辑推理 | 因果论证       | Because X, therefore Y.           |

### 3.4 分析与论证

提供论据后，必须进行分析，说明论据如何支持主题句：

| 步骤     | 说明           | 示例                                         |
| -------- | -------------- | -------------------------------------------- |
| 呈现论据 | 提供事实/数据  | The study found a 30% improvement.           |
| 解释含义 | 说明论据的意义 | This improvement indicates that...           |
| 联系论点 | 回扣文章主题   | Therefore, the proposed method is effective. |

### 3.5 段落间的过渡

| 关系 | 过渡词                                               |
| ---- | ---------------------------------------------------- |
| 递进 | furthermore, moreover, in addition, additionally     |
| 转折 | however, nevertheless, on the other hand, conversely |
| 因果 | therefore, consequently, as a result, thus           |
| 举例 | for example, for instance, specifically, namely      |
| 总结 | in summary, to conclude, overall, in brief           |
| 顺序 | first, second, finally, subsequently                 |
| 对比 | similarly, likewise, in contrast, whereas            |

## 4. 结尾段 (Conclusion)

### 4.1 结尾段的功能

1. **重述论点** (Restate Thesis)
2. **总结要点** (Summarize Main Points)
3. **升华主题** (Final Thought)

### 4.2 结尾段的写法

**方法一：总结式**

> In conclusion, while technology has brought remarkable convenience to modern life, it has also introduced new challenges that society must address. The key lies in harnessing technology's benefits while mitigating its risks through thoughtful regulation and ethical consideration.

**方法二：呼吁式**

> As we stand at the crossroads of technological advancement, it is imperative that we make conscious choices about the kind of digital future we want to create. The time to act is now.

**方法三：展望式**

> Looking ahead, the integration of AI into everyday life will only deepen. How we navigate this transformation will determine whether technology serves as a tool for empowerment or a source of division.

**方法四：引申式**

> The story of technology is ultimately a story about human choices. Every line of code, every algorithm, and every system reflects the values of its creators. The question is not whether technology will change us, but how we will choose to shape it.

### 4.3 结尾段的禁忌

| 禁忌                 | 说明                 |
| -------------------- | -------------------- |
| 引入新论点           | 结尾不应提出新的观点 |
| 简单重复             | 不要照搬开头段的原话 |
| 过于笼统             | 避免空泛的总结       |
| "In conclusion" 滥用 | 可用其他方式引出结尾 |

## 5. 特殊文章结构

### 5.1 五段式文章

```
第一段：开头段（引子 + 背景 + 主题句）
第二段：主体段1（论点1）
第三段：主体段2（论点2）
第四段：主体段3（论点3）
第五段：结尾段（重述 + 总结 + 升华）
```

### 5.2 对比型文章结构

| 结构     | 说明                             |
| -------- | -------------------------------- |
| 逐点对比 | 每段比较一个方面的异同           |
| 整体对比 | 先写A的所有方面，再写B的所有方面 |

**逐点对比：**

```
段落1：方面1 - A vs. B
段落2：方面2 - A vs. B
段落3：方面3 - A vs. B
```

**整体对比：**

```
段落1：A的方面1、2、3
段落2：B的方面1、2、3
段落3：A与B的对比总结
```

### 5.3 问题-解决型文章结构

```
开头段：提出问题
主体段1：分析原因
主体段2：提出解决方案
主体段3：评估方案可行性
结尾段：总结与展望
```

## 6. 写作常见问题

### 6.1 结构问题

| 问题       | 表现           | 解决方法              |
| ---------- | -------------- | --------------------- |
| 缺少主题句 | 段落主旨不明   | 每段开头写主题句      |
| 论据不足   | 观点缺乏支撑   | 每个论点至少2-3个论据 |
| 缺少分析   | 堆砌论据不分析 | 论据后加解释说明      |
| 过渡生硬   | 段落间缺乏衔接 | 使用过渡词            |
| 头重脚轻   | 开头过长       | 控制开头段比例        |

### 6.2 段落长度

| 段落类型 | 建议长度 | 说明     |
| -------- | -------- | -------- |
| 开头段   | 3-5 句   | 简洁有力 |
| 主体段   | 5-8 句   | 充分展开 |
| 结尾段   | 3-5 句   | 精炼总结 |
